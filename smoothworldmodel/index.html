Smooth World Model

🔒 Private  ·  viewable by anyone with this link  ·  not indexed

TL;DR

  • This repo's benchmark = speed, not quality. Batch 1, fixed prompt & seed → it reports latency (seconds), TTFT, throughput. Multiple runs are averaged only for timing stability — there is no quality score here.
  • Real world-model evaluation = quality. Because generation is stochastic, you draw N samples per input and aggregate with FVD (distribution-level) or best-of-N (per-sample). That code lives in cosmos-framework, not in this docs repo.
  • That \(N\) is exactly the "inference many times for a sample".

01 Model Architecture

Cosmos 3 is an omnimodal world model on a unified Mixture-of-Transformers (MoT) architecture combining an autoregressive (AR) transformer for reasoning with a diffusion transformer (DM) for multimodal generation.

Reasoner Mode

Language and visual-understanding tokens flow through causal self-attention for next-token prediction — perception, planning, and world reasoning.

Generator Mode

Noisy image / video / audio / action tokens are denoised through full attention, jointly generating coherent multimodal outputs.

Both modes share the same transformer, multimodal attention layers, and a unified 3D multi-dimensional rotary position embedding (mRoPE) encoding spatial and temporal structure across modalities — enabling consistent reasoning over images, videos, audio streams, and action trajectories.

★ Our idea

Treat the past window's latency & FLOPs as an extra condition, and let it inversely guide the next step's token budget: prune when recent cost is high, spend more when it is cheap — and later switch reason ⇄ generation mode. Open-loop map becomes a closed-loop, cost-aware pipeline.

$$ \begin{aligned} \hat{x}_{t+1} &= f_\theta(\,c\phantom{'},\ \epsilon\,)\phantom{\ \ \text{under } B_{t+1}} && \text{previous} \\[8pt] \hat{x}_{t+1} &= f_\theta(\,\color{#fb923c}{c'},\ \epsilon\,)\ \ \text{under }\color{#fb923c}{B_{t+1}} && \text{ours} \end{aligned} $$ $$ \color{#fb923c}{c'} = c \cup \color{#fb923c}{s_t}, \qquad \color{#fb923c}{s_t} = \{(\ell_\tau,\ \phi_\tau)\}_{\tau=t-w}^{\,t}, \qquad \color{#fb923c}{B_{t+1}} = \pi(\color{#fb923c}{s_t})\ \ (\,\color{#fb923c}{B_{t+1}}\!\downarrow \text{ as } \bar\ell_t\!\uparrow\,) $$

\(c'=c\cup s_t\) condition set unioned with the speed set · \(s_t\) latency \(\ell\) & FLOPs \(\phi\) over window \(w\) · \(\pi\) budget controller · \(B_{t+1}\) next-step token budget. \(f_\theta\) is realized in one of two surfaces:

a · Generator — flow-matching diffusion

encode → denoise (K=35, CFG) → decode $$ \begin{aligned} z_1 &\sim \mathcal{N}(0,I), \qquad c=\mathcal{E}(\text{prompt},\,\text{image}) \\[2pt] \tilde v_\theta &= v_\theta(z_t,t,\varnothing) + s\big[\,v_\theta(z_t,t,c) - v_\theta(z_t,t,\varnothing)\,\big] \\[2pt] z_{t-\Delta t} &= \mathrm{UniPC}\big(z_t,\ \tilde v_\theta,\ t\big),\quad t:\,1\!\to\!0 \\[2pt] \hat{x} &= \mathcal{D}(z_0) \end{aligned} $$

\(\mathcal{E},\mathcal{D}\) VAE encode/decode · \(v_\theta\) velocity (flow) · \(\varnothing\) null / negative prompt · \(s\) guidance scale. Code: num_inference_steps=35, guidance_scale=6.0, UniPCMultistepScheduler(flow_shift=10), 189 frames @ 24 fps.

b · Reasoner — autoregressive

next-token $$ x_k \sim p_\theta\big(x_k \mid x_{<k},\, c\big),\qquad c=[\,\text{text tokens},\ \mathcal{E}_{\text{vis}}(\text{image/video})\,],\quad k=1,2,\dots $$

Causal self-attention + 3D mRoPE; sample until EOS. Code: Cosmos3ReasonerForConditionalGeneration (vLLM).

04 Standard World-Model Eval Formulas

A world model's generation is stochastic: the same context with different noise/seed yields different valid futures. So the standard protocol samples \(N\) times per input, then aggregates one of two ways.

a · Distribution-level quality — FVD

The field standard for video world models. Pool generated vs. real clips and compare their I3D feature distributions:

Fréchet Video Distance $$ \mathrm{FVD} \;=\; \lVert \mu_r - \mu_g \rVert_2^{2} \;+\; \operatorname{Tr}\!\Big( \Sigma_r + \Sigma_g - 2\big(\Sigma_r \Sigma_g\big)^{1/2} \Big) $$

\(\mu_r,\Sigma_r\) = mean & covariance of real video features · \(\mu_g,\Sigma_g\) = same for generated · lower is better. FID is the single-image version of the same formula.

b · Per-sample, multimodal future — Best-of-N

Don't penalize the model for producing a different but valid future. Draw \(N\) samples per context \(c_i\) and keep the one closest to ground truth:

Best-of-N (min over samples) $$ \text{Score} \;=\; \frac{1}{M}\sum_{i=1}^{M}\; \min_{\,j=1,\dots,N}\; d\!\big(x_i,\ \hat{x}_i^{(j)}\big), \qquad \hat{x}_i^{(j)} \sim p_\theta(\,\cdot \mid c_i\,) $$

\(M\) = number of evaluation samples · \(N\) = generations per sample (the "inference many times") · \(c_i\) = conditioning context (init frames + action) · \(x_i\) = ground-truth future · \(\hat{x}_i^{(j)}\) = the \(j\)-th generation · \(d\) = PSNR / SSIM / LPIPS / feature distance. Typically \(N \in \{4,\dots,100\}\).

c · Reasoner (AR / VLM) — accuracy & pass@k

For discrete answers, sample \(n\) generations, count \(c\) correct, and report the chance that at least one of \(k\) is right:

pass@k $$ \text{pass@}k \;=\; \mathbb{E}_{\text{problems}}\!\left[\, 1 - \frac{\dbinom{n-c}{k}}{\dbinom{n}{k}} \,\right] $$

\(n\) = samples drawn per problem · \(c\) = number correct · \(k\) = budget (\(k=1\) recovers plain accuracy).

⚠ The key idea you asked about

"Inference many times for a sample" \(=\) the \(N\) in best-of-N (and the \(n\) in pass@k). It is essential for quality eval because the future is stochastic — but it is absent from the latency benchmark in this repo, which fixes the seed and measures only time.

05 One-line Summary

This repo's benchmark measures speed — one deterministic run, averaged for timing. Real world-model eval measures quality — N stochastic samples per input, aggregated via FVD (distribution) or best-of-N min-distance (per-sample), with pass@k for the reasoner — and lives in cosmos-framework.

Directly reusable: SRcn (compute-normalized success, SaiVLA-0) as our metric; Bench2Drive (already world-model-entangled) as our closed-loop harness with a wall-clock budget added; null baseline = the static profiling table — we must win only where it fails (dynamic load), via the residual \(r_t = \ell_t - T(B_t)\).

How the controller spends the budget — the knob \(b\) per architecture

The controller is architecture-blind: it only outputs an abstract budget level \(B_{t+1}=\pi(s_t)\) (an index into the feasible set). A thin per-model actuator turns that level into the model's real anytime knob \(b\); the table \(T(b)\) above links each level to its measured cost. The one idea behind every knob: spend fewer of the thing the model loops over. Pick a knob that is cheap to change per step, monotone in cost, and graceful in quality.

★ Accurate scores demand a totally free GPU

Because speed itself is the conditioning signal, the anchor (\(\lambda{=}0\), the "do-no-harm" reference) and the calibration table \(T(b)\) must be measured on a dedicated, idle, clock-locked GPU — no co-tenants, exclusive-process mode, fixed clocks (nvidia-smi -lgc, persistence on). Any uncontrolled background job makes the "idle" baseline noisy and corrupts the residual \(r_t=\ell_t-T(b)\) the controller integrates — you would be reacting to mystery load instead of the load you study. The \(\lambda{\neq}0\) runs then add a controlled, replayable antagonist on that same card: contention you injected, never contention you inherited.

09 Performance Targets

Reproduced scores are anchors — the top-right endpoint of a curve. We deliver the whole accuracy–efficiency curve, judged by three goals.

GoalMeaningCriterion
G1 · Do no harmidle GPU (λ=0, static table optimal) — never lose to the tablequality gap ≤ 1%

G3 · Significance≥3 seeds × ≥3 load schedulesgap passes significance test
Cost vector — "budget" is any measurable real cost, not just latency $$ c_t \;=\; \big(\, \ell_t,\ \phi_t,\ m_t,\ \rho_t,\ e_t,\ 1/\mathrm{fps}_t,\ \dots \,\big)\in\mathbb{R}^{d}, \qquad s_t=\{c_\tau\}_{\tau=t-w}^{t} $$

The fed-back signal \(s_t\) generalizes §02's \((\ell,\phi)\) to any runtime cost that represents real speed: wall-clock latency \(\ell\) · throughput \(1/\mathrm{fps}\) (FPS · tok/s) · peak GPU memory / KV-cache \(m\) · utilization / power \(\rho\) (W) · energy \(e\) (J) · hardware-independent FLOPs / MACs \(\phi\). The deadline \(\Delta t\) is just the \(\ell\)-axis of a general resource budget \(R=(\Delta t,\Phi_{\max},M_{\max},P_{\max},E_{\max},\dots)\).

Primary score — Q@ε at the resource-budget frontier $$ Q@\varepsilon \;=\; \max_{\pi}\ \text{Quality}(\pi) \quad \text{s.t.} \quad P\big(\, g(c_t) > R \,\big) \le \varepsilon $$

\(g(c_t)>R\) = the binding resource is exceeded — a latency miss \(\ell_t>\Delta t\) is the canonical instance; swap in a FLOPs cap, memory cap, power / energy cap, or a scalarized \(w^\top c_t>1\) and the same controller applies. \(\varepsilon\) = per-step violation tolerance. Report one \(Q@\varepsilon\) per (method · load · resource axis); global view: Pareto hypervolume over (quality, \(1/\text{cost}\)) across axes · external reference: SRcn (SaiVLA-0).

Constrained-benchmark formulation

$$ \text{standard:}\ \max_\pi S(\pi) \qquad\Longrightarrow\qquad \text{ours:}\ \max_\pi S(\pi)\ \ \text{s.t.}\ \ P\big(g(c_t) > R\big) \le \varepsilon \quad \big[\text{e.g. } \ell_t>\Delta t\big] $$
1 · Per-step, not aggregateconstraint on each step's cost \(c_t\) (latency, FLOPs, memory, power…), not the rollout aggregate — which step misses matters in closed loop (a miss mid-fall ≠ ten misses on flat ground)
2 · Physically enforcedmiss → repeat action / duplicate frame, consequence enters the dynamics — the constraint is part of the benchmark, not post-hoc filtering
3 · Curve + load axissweep \(B\) → tradeoff curve \(S(B)\); add \(\lambda(t)\) → static methods tie at \(\lambda{=}0\) (G1), lose at \(\lambda{\neq}0\) (G2). The drifting resource budget is why the \(s_t\) loop must exist — a fixed \(B\) is solved by table lookup

10 SOTA Plan — Smooth Controller

Design-panel recipe — zero-training wrappers on the reproduced checkpoints: DEADLINE-WM harness + BOLA-WM controller + VoC content-gate + CBF shield. The spine is borrowed from adaptive-bitrate video streaming.

The controller \(\pi\) — in: cost · out: next-inference strategy $$ \pi:\ \underbrace{s_t=\{c_\tau\}_{\tau=t-w}^{t}}_{\text{measured cost (input)}}\;\big[,\,g(\hat{x})\big]\ \longmapsto\ \underbrace{B_{t+1}}_{\text{strategy (output)}}\ \xrightarrow{\ \text{actuator}\ }\ b\quad(\text{e.g. AR token budget}) $$

Input = the cost window \(s_t\) (latency · FLOPs · GPU mem · power · energy…), optionally + content \(g(\hat{x})\). Output = the next step's strategy = budget level \(B_{t+1}\) → the per-architecture knob \(b\) (§08): AR token budget, diffusion denoising steps, MPC rollouts. The closed loop \(s_t\!\to\!\pi\!\to\!B_{t+1}\) is the contribution — and \(\pi\) can be built two ways, both ours:

\(\pi\) designHow it maps \(s_t \to B\)Cost · ceiling
Train-free \(\pi\) train-free
this paper's headline		closed-form online rules — EMA residual · BOLA debt-queue + CBF shield; reads \(s_t\), emits \(B\) by feasibility + argmax. No parameters.zero training, runs on the frozen checkpoint in ~1–2 days · ceiling = the static table's knob set
Trainable \(\pi_\theta\) train-based
"full version" · paper-2		a small learned policy (MLP / GRU) \(\pi_\theta(s_t,g(\hat{x}))\to B\), trained offline under randomized load \(\lambda(t)\) by RL (reward = quality − miss penalty) or imitation of the oracle; may also inject an \(s_t\)-embedding into the base model.costs training + a load simulator · higher ceiling — anticipates load, co-optimizes quality & budget

Start train-free for immediate SOTA-under-load; \(\pi_\theta\) is the higher-ceiling follow-up. The train-free spine in detail:

BOLA-WM — deadline-debt controller (from ABR, arXiv:1601.06748) $$ Q_{t+1} = \max\big(0,\ Q_t + g(c_t) - R\big), \qquad B_{t+1} = \arg\max_b\ \big[\, V\cdot \mathrm{quality}(b)\ -\ Q_t\cdot \mathrm{cost}(b) \,\big] $$

\(Q_t\) virtual resource-debt queue — is the online residual integrator (tracks load \(\lambda(t)\) with no prediction, no table) · \(g(c_t)-R\) per-step overage of the binding resource (latency \(\ell_t-D\) is the canonical instance; FLOPs / memory / power / energy debt work identically) · \(V\) quality-vs-safety knob. Prediction-free, table-free, ~1–2 days to wire, provably \(O(1/V)\) from optimal.
